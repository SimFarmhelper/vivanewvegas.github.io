Mod Managers to Avoid:

Vortex: Lacks many essential features (such as manual LO sorting), has an awful UI, and is very buggy. Uses Symlinks instead of a UVFS like MO2 (see here for a comparison). Avoid at all costs

NMM: Lacks many of the same features as Vortex, but gives the user slightly more control. Also prone to the same bugs such as failing to install a mod completely. Is no longer officially supported by Nexus, but I would still recommend it over Vortex. If you are for some reason use it over MO2, at least grab the version from GitHub

FOMM: Not too different than NMM, but I still recommend the GitHub release of NMM over it. If you are to use it, grab the Fork version. A big update for FOMM - Fork is in the works and will most likely be recommended over NMM once it releases

MO1: MO2, even though designed for 64bit games, works perfect for New Vegas

Other Tools to Avoid:

LOOT and automated conflict resolution (kind of): These shouldn't necessarily be completely avoided, but there are better, but not as easy, ways to manage large load orders. For more information on the topic, see the FIXING CONFLICTS section here

Nexus FNVLODGen: The Nexus version of FNVLODGen does not include the option to generate terrain LOD

GECK Powerup: No longer supported and is obsolete with GECK Extender

zEdit/zClean: Has no real advantages over xEdit and lacks the essential features/support that xEdit has

Ordenador/DDSopt: Breaks many textures by needlessly converting/adding alpha channels, cubemaps, and mipmaps

BethINI/Configator: Most INI changes are placebo at best, dangerous at worst. You really won't need any other INI tweaks than the ones in the guide
