NVSR: You may be wondering why NVSR isn't in the guide. NVSR has many broken settings, that when turned off essentially disable the features that remove the stuttering, making it pointless. There are also bugs with NVSR unrelated to settings, such as the really long fingers and bouncing NPCs. That being said, the following settings are broken in NVSR:

  • bManageFPS: Doesn't straight up limit FPS, but maintains a minimum FPS, so it starts to disable LOD and other things to maintain that FPS (paraphrased from RoyBatty)
  • bHookLightCriticalSections (Only fully broken on Windows 10): Bad implementation, broken by Windows 10 because the author relied on undefined behavior instead of
    following the API. Causes crashes every 15-ish minutes
  • bReplaceHeap: Outdated implementation, extremely unstable, not large address aware. Performs in function by increasing FPS, but at the cost of a much higher crash rate

ENBoost: NVR/NVGE has superior memory management and requires no tweaking, unlike ENBoost which needs a lot of system-specific tweaking to make your game not run worse

Zan AutoPurge Crash Protector: Unsafe/unstable, same function is performed safely by the .INI tweaks in the guide

PCB Hotkey: Unsafe

Save Cleaner: No, just no

Performance of the Gods (or pretty much all mods with "Performance" in the title): Performance gain is placebo, disabling random clutter items with less than 5 polygons isn't going to do anything. Potentially dangerous if they use Mark for Delete instead of Initially Disabled
